## Cell 6 — 定义数据读取与预处理函数

```python
CATEGORICAL = ['PULocationID', 'DOLocationID']

def read_data(url: str) -> pd.DataFrame:
    df = pd.read_parquet(url)
    df['duration'] = (df.tpep_dropoff_datetime - df.tpep_pickup_datetime).dt.total_seconds() / 60
    df = df[(df.duration >= 1) & (df.duration <= 60)].copy()
    df[CATEGORICAL] = df[CATEGORICAL].astype(str)
    return df
```

逐行拆解：


| 行                                                          | 说明                                                                                  |
| ---------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| `CATEGORICAL = ['PULocationID', 'DOLocationID']`           | 定义需要做 One-Hot 编码的两个分类特征：**上车地点 ID** 和 **下车地点 ID**                                   |
| `def read_data(url: str) -> pd.DataFrame:`                 | 函数签名，接收一个 URL 字符串，返回 DataFrame                                                      |
| `df = pd.read_parquet(url)`                                | 直接从远程 URL 读取 Parquet 文件到内存（pandas 会自动处理 HTTP 下载 + Parquet 解析）                       |
| `df['duration'] = ...`                                     | 计算行程时长（分钟）：下车时间 - 上车时间，先转为秒（`.dt.total_seconds()`），再除以 60                           |
| `df = df[(df.duration >= 1) & (df.duration <= 60)].copy()` | **去除异常值**：只保留 1~60 分钟的行程。太短的可能是取消/误触，太长的可能是数据错误。`.copy()` 避免 SettingWithCopyWarning |
| `df[CATEGORICAL] = df[CATEGORICAL].astype(str)`            | 把位置 ID 从数字转成字符串。**原因**：DictVectorizer 需要字符串类型才能做 One-Hot 编码；如果是数字，它会当作连续特征直接使用      |

## Cell 9 — 确定性采样（10%）

```python
SAMPLE_FRAC = 0.10
SEED = 42

len_train_before, len_val_before = len(df_train), len(df_val)

if SAMPLE_FRAC < 1.0:
    df_train = df_train.sample(frac=SAMPLE_FRAC, random_state=SEED)
    df_val   = df_val.sample(frac=SAMPLE_FRAC,   random_state=SEED)

print(f"Train size: {len_train_before:,} -> {len(df_train):,}")
print(f"Val   size: {len_val_before:,} -> {len(df_val):,}")
```


| 行                                       | 说明                                                            |
| --------------------------------------- | ------------------------------------------------------------- |
| `SAMPLE_FRAC = 0.10`                    | 采样比例 10%。改成 `1.0` 则不采样，使用全量数据                                 |
| `SEED = 42`                             | 随机种子，保证**可复现性**：每次运行采到完全相同的子集                                 |
| `len_train_before, len_val_before`      | 记录采样前的行数，方便前后对比                                               |
| `df.sample(frac=..., random_state=...)` | pandas 的随机采样方法。`frac=0.10` 表示取 10%，`random_state=42` 锁定随机数生成器 |
| **输出**                                  | Train: 2,898,906 → 289,891；Val: 2,938,060 → 293,806           |


**为什么要采样？** 在开发/调试阶段，用全量数据太慢。先在 10% 上快速验证逻辑和效果，确认没问题后再用全量跑最终结果。

## Cell 12 — 特征工程：One-Hot Encoding

```python
train_dicts = df_train[CATEGORICAL].to_dict(orient='records')

dv = DictVectorizer()

X_train = dv.fit_transform(train_dicts)

y_train = df_train['duration'].values

print('Feature matrix (train):', X_train.shape)
```

逐行拆解：


| 行                                                 | 说明                                                                                        |
| ------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| `df_train[CATEGORICAL].to_dict(orient='records')` | 把 DataFrame 的两列转成字典列表。例如：`[{'PULocationID': '186', 'DOLocationID': '79'}, ...]`。每一行变成一个字典 |
| `dv = DictVectorizer()`                           | 初始化 DictVectorizer。它会自动把字典中的**字符串值做 One-Hot 编码**，**数字值保持原样**                              |
| `X_train = dv.fit_transform(train_dicts)`         | **fit**：扫描所有字典，建立"类别 → 列索引"的映射表。**transform**：把每个字典转成稀疏向量。两步合一                            |
| `y_train = df_train['duration'].values`           | 提取目标变量（行程时长），转成 numpy 数组                                                                  |
| **输出**                                            | 特征矩阵形状 `(289891, 491)`：289,891 个样本，491 个 One-Hot 特征（即训练集中共出现了 491 种不同的 PU/DO 位置组合）        |


**关键概念 — 为什么用稀疏矩阵？** 491 列中每行只有 2 个非零值（一个 PU，一个 DO），其余全是 0。稀疏矩阵只存非零元素，极大节省内存。

```python
feature_names = dv.get_feature_names_out()

row = 0

decoded = [(feature_names[i], X_train[row, i]) for i in nz_cols]
pd.DataFrame(decoded, columns=['feature', 'value'])
```


| 行                            | 说明                                                                                             |

| `decoded = [...]`            | 把列索引翻译成"特征名 + 值"的元组列表                                                                          |
| **输出**                       | 第 0 行有两个非零特征：`DOLocationID=261`（值 1.0）和 `PULocationID=163`（值 1.0）。这就是 One-Hot 编码：对应位置为 1，其余为 0 |

## Cell 17 — 训练线性回归模型

```python
lr = LinearRegression()

start_train = time.time()
lr.fit(X_train, y_train)
end_train = time.time()
train_duration = end_train - start_train

start_pred = time.time()
y_pred_train = lr.predict(X_train)
end_pred = time.time()
pred_duration = end_pred - start_pred

rmse_train = root_mean_squared_error(y_train, y_pred_train)

print(f"Train RMSE: {rmse_train:.3f}")
print(f"Training time: {train_duration:.2f} seconds")
print(f"Prediction time: {pred_duration:.2f} seconds")
```


| 行                                                | 说明                                                              |
| ------------------------------------------------ | --------------------------------------------------------------- |
| `LinearRegression()`                             | 初始化线性回归模型（普通最小二乘法，无正则化）                                         |
| `time.time()` 包裹 `lr.fit(...)`                   | 计时训练过程。`fit` 在内部求解 X^T X β = X^T y                              |
| `lr.predict(X_train)`                            | 用训练好的模型对训练集做预测                                                  |
| `root_mean_squared_error(y_train, y_pred_train)` | 计算 RMSE = √(Σ(y_actual - y_pred)² / n)，衡量预测误差的平均大小，单位与 y 相同（分钟） |
| **输出**                                           | Train RMSE ≈ 7.924 分钟，训练耗时 0.76 秒，预测几乎瞬间完成                      |


**注意**：这里只是 Train RMSE，不能代表泛化能力。需要看 Validation RMSE 才能评估过拟合情况。

## Cell 21 — 验证集评估

```python
val_dicts = df_val[CATEGORICAL].to_dict(orient='records')
X_val = dv.transform(val_dicts)
y_val = df_val['duration'].values

y_pred_val = lr.predict(X_val)
rmse_val = root_mean_squared_error(y_val, y_pred_val)
print(f"Validation RMSE: {rmse_val:.3f}")
```


| 行                                               | 说明                                                                           |
| ----------------------------------------------- | ---------------------------------------------------------------------------- |
| `df_val[CATEGORICAL].to_dict(orient='records')` | 和训练集一样，把验证集的分类特征转成字典列表                                                       |
| `dv.transform(val_dicts)`                       | **只用 transform，不用 fit_transform！** 使用训练集建立的映射表来转换验证集。如果验证集出现训练集没见过的类别，会被自动忽略 |
| `X_val = ...`                                   | 验证集的特征矩阵                                                                     |
| `y_val = ...`                                   | 验证集的真实行程时长                                                                   |
| `lr.predict(X_val)`                             | 用训练好的模型预测验证集                                                                 |
| **输出**                                          | Validation RMSE ≈ 8.113 分钟                                                   |


**关键点**：

- Train RMSE (7.924) vs Val RMSE (8.113)：差距很小（约 0.19），说明**没有明显过拟合**
- 但 RMSE ≈ 8 分钟意味着平均预测误差约 8 分钟，对于平均 14 分钟的行程来说还有很大提升空间

## 整体流程总结

```
远程 Parquet 文件
       │
       ▼
  read_data()
  ├─ 读取 parquet
  ├─ 计算 duration（分钟）
  ├─ 过滤异常值（1~60 min）
  └─ 分类列转字符串
       │
       ▼
  确定性采样 10%（seed=42）
       │
       ▼
  DictVectorizer
  ├─ fit_transform（训练集）→ 稀疏矩阵 X_train
  └─ transform（验证集）    → 稀疏矩阵 X_val
       │
       ▼
  LinearRegression
  ├─ fit(X_train, y_train)
  ├─ predict(X_train) → Train RMSE ≈ 7.92
  └─ predict(X_val)   → Val RMSE   ≈ 8.11
       │
       ▼
  可视化：actual vs predicted 直方图
```

### 核心概念速查


| 概念                   | 本 Notebook 中的体现                               |
| -------------------- | --------------------------------------------- |
| **Parquet 格式**       | 列式存储，压缩率高，读大数据快                               |
| **One-Hot Encoding** | DictVectorizer 把 PU/DOLocationID 转成 491 维稀疏向量 |
| **稀疏矩阵**             | 每行仅 2 个非零值，节省内存                               |
| **确定性采样**            | `random_state=42` 保证可复现                       |
| **Train/Val 分离**     | 时间维度切分（1 月 vs 2 月），验证集只 transform 不 fit       |
| **RMSE**             | 衡量预测误差，单位与目标变量一致（分钟）                          |
| **过拟合检测**            | 比较 Train RMSE 和 Val RMSE 的差距                  |
